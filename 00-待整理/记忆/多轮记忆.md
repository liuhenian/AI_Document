LangGraph 实现多轮对话，本质上靠的是 **状态图 + 检查点 + 线程隔离** 这三板斧。它让每次对话调用不再是独立的请求，而是同一张有状态图上的连续执行。

下面我拆成 4 步，给你一个可以直接跑起来的实现。

---

### 0.1.1 定义状态：把对话历史放进图里

多轮对话的核心就是消息历史。在 LangGraph 中，你需要定义一个包含 `messages` 的状态类：

```python
from typing import TypedDict, Annotated
from langgraph.graph.message import add_messages
from langchain_core.messages import AnyMessage

class State(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    # 你可以加任意其他字段，比如 user_id, context...
```

- `add_messages` 是一个 **reducer**，它保证每次节点返回的 `messages` 会**追加**到现有列表，而不是覆盖。这样历史就自然累积了。

---

### 0.1.2 构建图：一个简单的聊天节点

```python
from langgraph.graph import StateGraph, START, END
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o")

def chatbot(state: State):
    # 调用模型，传入整个消息历史
    response = model.invoke(state["messages"])
    # 返回的 {"messages": [response]} 会自动追加到历史
    return {"messages": [response]}

graph_builder = StateGraph(State)
graph_builder.add_node("chatbot", chatbot)
graph_builder.add_edge(START, "chatbot")
graph_builder.add_edge("chatbot", END)
```

到这里，这还只是一个无状态的图，每次运行都会重新开始。关键在下一步。

---

### 0.1.3 挂载检查点，用 thread_id 保持状态

这是 LangGraph 多轮对话的**灵魂**：

```python
from langgraph.checkpoint.memory import MemorySaver

# 创建一个检查点存储器（生产环境可换 SqliteSaver/PostgresSaver）
memory = MemorySaver()

# 编译图时传入 checkpointer
app = graph_builder.compile(checkpointer=memory)
```

现在，每次调用时传入相同的 `thread_id`，LangGraph 就会自动从上次保存的状态继续执行。

```python
config = {"configurable": {"thread_id": "user-session-123"}}

# 第一轮
response1 = app.invoke(
    {"messages": [{"role": "user", "content": "我叫小明"}]},
    config
)
print(response1["messages"][-1].content)  # 模型回复

# 第二轮，直接发新消息，不需要再传历史
response2 = app.invoke(
    {"messages": [{"role": "user", "content": "我叫什么名字？"}]},
    config
)
print(response2["messages"][-1].content)  # 模型会回答 "小明"
```

**发生了什么？**
- `invoke` 时，LangGraph 根据 `thread_id` 查找检查点，加载之前的 `State`（包含历史消息）。
- 新消息通过 `add_messages` 追加进去。
- 节点 `chatbot` 看到的是 **完整的消息历史**（第一轮的两条 + 第二轮的新用户消息）。
- 执行结束后，新状态自动写入检查点，等待下一次调用。

这样，多个 HTTP 请求、跨天、甚至服务重启后（如果用了持久化存储），对话都能接上。

### 0.1.4 支持工具调用 / Agent 循环的多轮对话

真实 Agent 往往需要一个“思考-行动-观察”循环。LangGraph 中只要加一个条件边：

```python
from langgraph.prebuilt import ToolNode
from langchain_core.tools import tool

@tool
def get_weather(city: str):
    """查询天气"""
    return f"{city}：晴天 25°C"

tools = [get_weather]
model_with_tools = model.bind_tools(tools)

def chatbot(state: State):
    return {"messages": [model_with_tools.invoke(state["messages"])]}

def should_continue(state: State):
    last_message = state["messages"][-1]
    if last_message.tool_calls:
        return "tools"
    return END

graph_builder.add_node("tools", ToolNode(tools))
graph_builder.add_conditional_edges("chatbot", should_continue, {"tools": "tools", END: END})
graph_builder.add_edge("tools", "chatbot")  # 工具执行完回到 chatbot 继续思考

app = graph_builder.compile(checkpointer=memory)
```

一次用户消息可能触发多次模型调用和工具调用，这些中间步骤的状态也都会保存在检查点中。下一轮用户发消息时，整个历史（包括之前的工具交互）都在。

---

### 0.1.5 高级：人工介入（Human-in-the-loop）

如果你希望在某个节点暂停等待用户输入（比如确认敏感操作），可以使用**中断**：

```python
app = graph_builder.compile(
    checkpointer=memory,
    interrupt_before=["tools"]  # 在执行工具前挂起
)

# 用户发消息后，图会停在 tools 之前
response = app.invoke({"messages": [user_input]}, config)
# 此时 response 里是到中断点之前的状态

# 人工确认后，用 Command(resume=...) 继续执行
from langgraph.types import Command
app.invoke(Command(resume={"some_override": value}), config)
```

这对于多轮对话中的审批、修正等场景非常实用。

---

### 0.1.6 总结：LangGraph 多轮对话的核心要素

| 要素                         | 作用               |
| -------------------------- | ---------------- |
| **State 中的 messages 字段**   | 累积全量对话历史         |
| **`add_messages` reducer** | 自动追加而非覆盖         |
| **Checkpointer**           | 持久化状态（内存/数据库）    |
| **`thread_id`**            | 隔离不同会话，加载对应状态    |
| **条件边/循环**                 | 支持工具调用、多步推理等复杂对话 |
|                            |                  |

你只需要在每次请求时带上相同的 `config`，LangGraph 就把“无状态 API 调用”变成了“有状态的连续对话”。这种设计让多轮对话、多步 Agent、中断等待等高级模式变得统一且优雅。
